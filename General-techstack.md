Tech Stack Reference Core Framework - Next.js 16 (App Router) - Full-stack React framework - React 18 - UI library - JavaScript (ES6+) - No TypeScript, uses jsconfig.json for path aliases (@/*) --- Styling & UI - Tailwind CSS 3.4 - Utility-first CSS - shadcn/ui - Pre-built accessible components (via CLI) - CVA (Class Variance Authority) - Component variant management - clsx + tailwind-merge - Class name utilities - PostCSS + Autoprefixer - CSS processing - Critters - Critical CSS extraction Icons: - lucide-react - @tabler/icons-react --- Database - Firebase/Firestore - NoSQL document database - firebase-admin - Server-side SDK - Firebase Storage - File uploads with public URLs - Custom abstraction layer (lib/db.js) - No ORM, direct Firestore operations --- Authentication - Custom session-based auth - Base64 encoded cookies (7-day expiration) - WebAuthn Passkeys - Passwordless authentication support - Role-based access control - admin, client, team, user roles - Middleware protection - Route-level access enforcement --- API Layer - REST API - Next.js App Router API routes (/app/api/**/route.js) - Standard HTTP methods - GET, POST, PUT, DELETE - JSON request/response - No tRPC or GraphQL --- Forms & Validation - Native form handling - No React Hook Form - Custom validation - Regex-based, no Zod/Yup - Server Actions - Form submissions to API endpoints --- State Management - React Context API - Session provider for auth state - No Redux/Zustand - Leverages Server Components for data fetching --- Animation - Framer Motion 11 - Production animations - GSAP 3.13 - Advanced animation toolkit - tsparticles - Particle effects --- Payments - Stripe - Checkout sessions, webhooks, billing portal --- PDF Generation - Puppeteer - Full browser automation (development) - puppeteer-core + @sparticuz/chromium - Serverless PDF generation - Dual-mode execution - Detects Vercel environment automatically --- Email - Pluggable architecture (lib/email/send.js) - Console adapter (dev) - Logs to terminal - Ready for SendGrid/Resend - Integration patterns in place - Template system - Reusable email templates --- AI Integration - OpenAI SDK - GPT + DALL-E + Vision API - Custom portrait generator - AI image generation with payment integration --- Data & Utilities - date-fns - Date manipulation - uuid - ID generation - recharts - Data visualization/charts - react-big-calendar - Calendar component - sharp - Image processing --- Localization System Architecture: Custom i18n solution (no external library) File Structure: localization/ ├── index.js # Central export hub └── en/ ├── common.json # Company, nav, UI strings ├── cta.json # Call-to-action content ├── pages/ # Page-specific content (13 files) ├── components/ # Component content (6 files) ├── pricing/ # Pricing tier data └── seo/ # Metadata & keywords Usage Pattern - Direct Import: import { homeContent, cta, common } from "../localization"; export default function Page() { return <h1>{homeContent.hero.title}</h1>; } Usage Pattern - Helper Functions: import { t, getCta, getContent } from "lib/i18n"; // With interpolation t('common.company.copyright', { vars: { year: 2025 } }) // → "© 2025 BEIRUX.
